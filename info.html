About RedRose OS

RedRose is a security-focused live distribution based on an Arch rolling base. Below you will find features, changelogs and a concise comparison between Snapshot 0.2 and 0.1.

What is RedRose?

RedRose provides privacy-oriented, ephemeral live images for testing, research and incident response workflows. Snapshots are trimmed, curated images designed to boot quickly and provide a stable security toolkit for investigative tasks.

Core Features

  • Live, RAM-resident operation — leaves no trace on the host when removed.
  • Optimised toolset — curated security utilities pre-configured for lab workflows.
  • Performance — Arch rolling base and kernel tweaks for fast boot and runtime.
  • Custom visuals — tailored icons, splash assets and a black-red default wallpaper on 0.2.

Changelogs

Snapshot 0.2 — Changelog (English)

System & Kernel

  • GZ Mode integrated: gz moved to /usr/local/bin, permissions enforced via profiledef.sh.
  • Build cleanup routines added for work folder and /proc mount issues.

Interface & Terminal

  • Terminal identity: new minimalist red-white [RedRose] $ prompt running on zsh.
  • Font update: switched to Noto Sans and JetBrains Mono Nerd Font to fix rendering issues.

Visual

  • New GRUB-friendly 8-bit splash, custom icons (redrose_icon.png) and default black-red wallpaper.
Snapshot 0.1 — Changelog (English)

Snapshot 0.1 is the initial live image snapshot. It contains a compact set of tools for quick forensic and testing tasks. Notable points:

  • Live-only image (no installer)
  • Core utilities such as nmap included
  • Designed for hardware compatibility and lower-size footprint
